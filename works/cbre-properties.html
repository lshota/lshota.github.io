CBRE Japan 物件検索・会員管理 Web システム

世界最大の事業用不動産サービス会社 CBRE の物件検索・会員管理システムのフロントエンドリニューアル。約26万件の物件データを扱う大規模サイトを Next.js 14 + Turborepo モノレポで構築。

  • Next.js 14
  • TypeScript 5
  • Turborepo
  • AWS Lambda
  • Cognito
  • ElastiCache Redis
CBRE

概要

オフィス・店舗・物流施設等の物件検索、会員向けマイページ、コラム・イベント配信を担うユーザー向けサイトと管理画面のフロントエンドリニューアルに参画。 Turborepo モノレポ環境にて、フロントエンド実装と Next.js API 層の設計・組み込みを担当しました（バックエンド API・DB・インフラは別チームが担当）。

技術的課題

  • Next.js 14 の SSR/ISR を AWS Lambda 上で稼働させる構成のため、コンテナ間でキャッシュが共有されず、ページ再生成の負荷が集中する
  • ユーザー向け・管理者向けで別々の Cognito User Pool を利用しつつ、JWT 検証と API 権限制御を共通ライブラリで統一する必要がある
  • 物件種別ごとに異なる検索条件（エリア・路線・坪数・賃料等）を扱う検索 UI・一覧表示の実装

主な担当

  • Turborepo + pnpm ワークスペースで、ユーザーサイト・管理画面・共有パッケージの型定義を TypeScript strict で共通化
  • Next.js 14（App Router）による物件検索・マイページ・お問い合わせフォームの実装と、withAuth / withPublicAccess による API 認可層の共通化
  • ユーザー向け・管理者向けの 2 つの Cognito User Pool を前提とした認証基盤の設計・実装（httpOnly Cookie + JWT 検証、ミドルウェアによるルート保護、トークン期限切れ時のリダイレクト処理）
  • @neshca/cache-handler + ElastiCache Redis による Next.js Data Cache の Lambda 間共有と、管理画面更新時にフロントのキャッシュを無効化する On-Demand Revalidation の組み込み

成果

  • 本番稼働中。約 26 万件の物件データを扱う検索・一覧表示を安定提供
  • 8名体制（フロントエンド6名＋DB・API 2名）でのコードレビュー・型共有・キャッシュ設計を踏まえたチーム開発を経験
